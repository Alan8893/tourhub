# TH-0103 — Operational Write Audit Coverage

## Status

DONE

## Goal

Extend semantic actor-aware audit coverage to the remaining approved operational write paths without ORM-wide auditing or a change to the accepted single-club architecture.

## Delivered

- Product create/update actions with bounded snapshots and no-op suppression.
- Successful Product/Recipe CSV apply actions with aggregate metadata only.
- Purchase-list and purchase-checklist generation, responsible-person updates, and checklist item progress.
- Recipe equipment requirement create/update/delete.
- Project EquipmentList generation and manual add/update/delete operations.
- Successful Project focused/consolidated/package documents and legacy purchase-list PDF/Excel generation.
- Russian Administrator Audit labels, entities, filtering, and 360 px overflow acceptance.

## Transaction and safety rules

- Domain-write events are recorded before the owning commit and roll back with the domain mutation.
- Services invoked with `commit=False` append AuditEvents to the caller-owned Project preparation transaction.
- No-op edits do not create AuditEvents.
- Successful document-generation events commit after generation and before response delivery.
- CSV content, import row values, generated bytes/text, filenames, arbitrary request bodies, credentials, tokens, SMTP internals, and exception details are excluded.
- Automatic ORM-wide auditing remains rejected.
