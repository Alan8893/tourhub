# Documentation First Rule

Before any code changes, architectural changes, or domain decisions, the current documentation must be reviewed.

Required documents:

- PROJECT_CONTEXT.md;
- ARCHITECTURE.md;
- DOMAIN.md;
- DEVELOPMENT_RULES.md;
- relevant ADR documents;
- active task documentation.

Only after documentation analysis may implementation begin.

If business rules change, documentation must be updated before or together with code changes.

# Existing rules remain unchanged

Документация является частью продукта.

При изменении архитектуры обновляются:

- ARCHITECTURE.md;
- PROJECT_CONTEXT.md.

При изменении предметной области обновляется:

- DOMAIN.md.

При изменении процесса разработки обновляется:

- DEVELOPMENT_RULES.md.
